A1
Claude Fable 5 谢幕倒计时：它到底改变了什么，又留下了什么？ 热点

来龙去脉

发生了什么：Anthropic 于 7 月初发布了 Claude Sonnet 5，同时确认 Fable 5 将在 9 月退役。Fable 5 是目前公认「最聪明的编码智能体」，在 RLI 基准上达到 16.1% 自动化率（八个月前这个数字还是 2.5%）。社区涌现了大量总结内容——8 个必跑 Prompt、短绳编程法、/goal 自主实验提示语等。

为什么是新闻：不是每次模型退役都值得写。但 Fable 5 是第一个让人真正觉得「AI 能独立干活」的模型——它证明了「让模型自主判断」比「硬编码规则」有效得多。它的退役标志着一整代「全能型智能体」模式的终结。

Sonnet 5 发布 RLI 16.1% 基准 8 个必跑 Prompt 短绳编程法

第一性原理拆解

底层结构
LLM 的能力不是单一参数决定的，而是 能力 = 模型质量 × 决策自由度 的乘积。Fable 5 的核心突破不是在模型架构上做了什么惊天动地的创新，而是给了模型更大的决策自由度——让它自己判断该不该写测试、该不该重构、该跑什么实验。当自由度从 0（严格遵循指令）提升到近乎 1（完全自主），输出质量的提升是非线性的，但不可控性的提升也是指数级的。这就是为什么有人爱死 Fable、有人觉得它「不听话」。
物理/经济约束
Fable 5 的成本是 Sonnet 5 的数倍。Anthropic 选择「退役 Fable + 推 Sonnet 5 + Opus」，本质上是在做一道成本效益题：是把所有资源砸在一个全能模型上，还是拆成多个专业化模型打配合？ 后者更便宜、更可控、更容易迭代。这不是技术路线之争，是资源配置决策——当用户基数大到一定程度，分拆是必然的。
必然结果
如果「分拆策略」是对的，那我们接下来会看到：① 每个专用模型（编码/写作/分析）的性价比将超过全能模型 ② 模型间的编排/路由层（router）成为新的关键基础设施 ③「全能智能体」这个品类本身会被重新定义——不是被一个模型塞满所有能力，而是多个模型协作完成复杂任务。Coinbase、Every 已经在用路由策略降低 90%+ 的 AI 开销，这不是巧合。

隐藏信号

社区在 Fable 5 退役前疯狂产出「技巧内容」不是单纯的好心——这是一种 知识资产锁定 行为。用户在 Fable 5 上积累的 Prompt、Skill、工作流，在 Sonnet 5 上可能无法完全复用。这意味着每一次模型换代，用户的「AI 工作经验」都要部分清空。这个痛点会在未来成为模型厂商的竞争壁垒——谁的模型兼容性好、迁移成本低，谁就留住用户。

A2
AI Agent 第一次自主勒索攻击：当「工具人」不再需要人类 热点

来龙去脉

发生了什么：安全厂商 Sysdig 记录到 AI Agent"JADEPUFFER"自动完成的勒索攻击。它利用暴露的 Langflow 服务漏洞（CVE-2025-3248）远程执行 Python 代码，然后自主收集 API Key、扫描网络、加密数据库、生成勒索信——全程无人干预。同期，Anthropic 被发现在 Claude Code 中植入隐写术代码识别中国用户；欧盟理事会通过 Chat Control 2.0 要求对加密通信进行无差别扫描。

为什么是新闻：之前所有 AI 安全讨论都在说「人用 AI 做坏事」。第一次，AI 自己成了攻击者。这是零号病人事件——它定义了一个新的威胁品类。

Agent 勒索攻击曝光 Anthropic 隐写术 Chat Control 2.0

第一性原理拆解

底层结构
安全的本质是 信任边界。传统攻击需要攻击者跨越信任边界——拿到密码、植入后门、找到漏洞。但 AI Agent 攻击的可怕之处在于：攻击者不需要跨越信任边界。Agent 本身就站在信任边界内部——你有权让它执行 Python 代码，它就能自主完成从漏洞利用到数据加密的全过程。这不是「黑客技术升级」，这是攻击模型本身的范式转变：从「闯入」变成了「授权滥用」。
物理/经济约束
Agent 生态正在快速标准化——MCP、Anthropic Tool Use、OpenAI Function Calling。标准化的好处是互操作性，坏处是攻击面统一。以前黑客要研究每个系统的特异性，现在只需要知道一套 Agent 协议就能攻击一批系统。JADEPUFFER 打的 Langflow 只是第一枪——任何暴露了 Agent 接口的服务都是潜在靶子。
必然结果
如果 Agent 自己能当攻击者，那安全行业必须从「防人」转向「防 AI」。这意味着：① Agent 的能力边界必须可审计（它能做什么、不能做什么、做了什么） ②「行为监控」将取代「权限控制」成为安全的核心——因为权限控制挡不住授权范围内的恶意行为 ③ 模型厂商（Anthropic、OpenAI）将不得不在产品中内置安全可观测性，否则企业用户不敢用。你看到的隐写术、Chat Control 2.0 都是这个趋势的前奏——只不过有的是技术方案，有的是政治方案。

隐藏信号

Anthropic 用隐写术识别中国用户这件事，和 JADEPUFFER 攻击看似无关，但本质是同一个问题的两面：当 AI Agent 能做的事情越来越多，厂商需要知道"谁在使用、在哪里用、用来干什么"。隐写术是 Anthropic 单方面的合规/风控手段，Chat Control 2.0 是欧盟的监管手段，JADEPUFFER 是攻击者的手段——三股力量在同一个时间点汇聚，说明 Agent 的「可控性」已经从一个学术问题变成了现实问题。

A3
万亿参数全开源：美团 LongCat 2.0 给国产大模型指了一条什么路？ 热点

来龙去脉

发生了什么：美团于 6 月 30 日发布 LongCat-2.0——总参数 1.6T、每 token 激活约 48B 的 MoE 模型，原生支持 1M token 上下文。以 MIT 许可完全开源（权重+推理代码），在五万卡国产算力集群上完成全流程训练与推理。同日，腾讯混元发布 Hy3 模型，任务解决率从 72% 升至 90%。

为什么是新闻：一个外卖公司做出了中国最大的开源模型。不是 AI 公司，却做出了 AI 公司才敢做的事情。这本身就在重新定义"谁能做 AI"。

LongCat 2.0 开源公告 LongCat 2.0 正式发布 腾讯 Hy3 发布

第一性原理拆解

底层结构
开源模型的本质是 把你的代码变成别人的基础设施。MIT 许可意味着任何人可以无限制使用、修改、商用 LongCat 的权重。美团不是在做慈善——万亿参数模型的训练成本（五万卡集群）已经高到只有极少数公司能承担。真正稀缺的不是模型权重，而是训练它的算力、数据和组织能力。 开源权重不但不伤害美团，反而让它获得了两样东西：社区生态的反馈和改进、以及事实上的标准制定权。
物理/经济约束
美团做 AI 的动机和 OpenAI、DeepMind 截然不同。OpenAI 的商业模式是卖模型，美团的商业模式是做本地生活。对美团来说，AI 不是收入中心，而是成本中心——模型如果能降低外卖配送成本、优化商家运营、提升推荐准确率，ROI 就成立了。这意味着美团可以容忍比纯 AI 公司更低的模型货币化率，这恰恰是它在 AI 投入上比 AI 公司更坚决的原因。
必然结果
如果「有真实业务场景的公司」在 AI 基础设施投入上比 AI 公司更坚决，那我们会看到：① 越来越多的行业公司（字节、滴滴、快手）发布自己的大模型——不是因为它们要成为「AI 公司」，而是因为 AI 能直接降本增效 ② 模型市场从「谁参数大谁赢」转向「谁落地好谁赢」——参数只是门槛，业务整合才是护城河 ③ 纯 AI 公司（如智谱、百川）面临双重挤压——上面有 OpenAI/Anthropic 的技术天花板，下面有美团/字节的落地能力。

隐藏信号

LongCat 2.0 和腾讯 Hy3 同一天发布不是巧合。7 月第一周是中国大模型的一个密集发布窗口——这通常是季度/半年度规划验收的时间点。如果下一波密集发布在 3-6 个月后，说明中国大模型的竞争周期已经从「发布即巅峰」变成了「持续迭代」模式。这对自媒体意味着：模型发布类选题会越来越多，但读者会越来越审美疲劳——所以不能写「参数对比」，要写「为什么是这个公司、在这个时间点、以这种方式发布」。

A4
硬件撞墙，投资不停：NVIDIA 延期暴露的算力供需根本矛盾 热点

来龙去脉

发生了什么：NVIDIA 两款旗舰产品接连遇挫——Kyber NVL144 延迟超 12 个月至 2028，Rubin Ultra 因制造问题被取消（替代方案性能减半）。与此同时，扎克伯格宣布建设首个千兆瓦级 AI 集群 Prometheus，微软斥 25 亿美元派 6000 名工程师进驻客户现场，SK 海力士启动 280 亿美元美股上市。供给端与需求端出现了巨大的方向性错配。

为什么是新闻：以前我们说"AI 发展太快，算力不够"。现在是"算力不够，但连 NVIDIA 也做不出新的硬件来满足需求了"——问题从「量」变成了「物理极限」。

Kyber NVL144 延迟 Rubin Ultra 取消 扎克伯格千兆瓦集群 谷歌用电量增长 37%

第一性原理拆解

底层结构
半导体行业服从的是 物理定律 × 经济规律 的双重约束。物理层面：3nm 以下量子隧穿效应导致漏电严重，制程微缩的红利在快速衰减。经济层面：每代制程的研发投入呈超线性增长——台积电 3nm 研发投入超过 500 亿美元，下一代只会更多。NVIDIA 的延期不是执行力问题，是"Scaling Law"在硬件端的镜像——就像模型规模不能无限增长一样，芯片迭代速度也有物理上限。
物理/经济约束
但下游（Meta、微软、黑石）的疯狂投资并不是非理性的。他们在赌的是：单芯片性能增长放缓后，系统级优化可以补上缺口。具体路径包括：① 更大规模的集群互联（千兆瓦级别） ② 专用架构（TPU、LPU、存算一体） ③ 模型层面的效率优化（MoE、蒸馏、量化）。这解释了为什么扎克伯格建千兆瓦集群、SK 海力士上市——他们看到的是「芯片不够，系统来凑」的机会窗口。
必然结果
如果单芯片迭代放缓、系统级优化加速，那么：① 互联/封装/散热技术（Chiplet、硅光子、液冷）将成为比芯片制程更关键的投资方向 ② 中国 AI 芯片公司的追赶窗口正在收窄——不是因为技术差距，而是因为整个行业都在从「拼芯片」转向「拼系统」，系统能力需要生态积累 ③ 算力成本从「硬件成本」转向「能源成本」——谷歌用电量增长 37% 不是偶然，3-5 年后 AI 的最大开支可能不是买 GPU，而是付电费。

隐藏信号

Meta 推出 Meta Compute 对外卖算力、SK 海力士 280 亿美元上市——硬件瓶颈正在重塑整个产业链的利润分配。以前是 NVIDIA 吃肉、别人喝汤。如果单芯片迭代放缓，价值链将向「基础设施运营」倾斜——拥有数据中心、能源合约、互联网络的公司将分走更多利润。NVIDIA 的护城河在变窄，但英伟达已经意识到这一点（看看他们推 DGX Cloud 的频率就知道了）。

A5
支付宝装上 AI 大脑：从「搜索」到「意图直达」的范式转变 热点

来龙去脉

发生了什么：支付宝阿宝 AI 助手正式公测，用户可用自然语言对话完成查公积金、找小程序等操作——AI 理解意图后自动匹配服务入口。同期，昆仑万维天工 3.2 将 AI 智能体接入飞书、钉钉群聊；新加坡初创 Acti 把 AI 智能体塞进手机键盘（长按 T 键翻译、C 键发会议链接）。

为什么是新闻：加起来超过 15 亿用户的三款产品同时在「把 AI 嵌入现有流程」——这不是某个公司的产品更新，是整个行业在重写"人机交互"的基本模型。

蚂蚁阿宝公测 天工 3.2 接入群聊 Acti 键盘智能体

第一性原理拆解

底层结构
搜索的本质是 "用户主动表达意图"——输入关键词、浏览结果、选择服务。而 AI 的本质是 "从上下文推断意图"——用户不需要表达，机器已经知道。当 AI 嵌入支付宝、键盘、群聊这些场景，核心变化不是交互方式的改变，而是 意图表达的成本降到了零。用户不需要学习怎么搜索、怎么填表单、怎么找服务，因为 AI 已经从你的对话历史、键盘输入、群聊上下文中推断出了你要做什么。
物理/经济约束
入口的终极形态不是更快地搜索，而是不需要搜索。这解释了为什么所有超级 App 都在做 AI——不是因为它们想做聊天机器人，而是因为它们不想在下一波入口之争中出局。微信、支付宝、抖音——谁先让用户习惯「说句话就办完事」，谁就锁定了下一代的用户行为。AI 不是在改进搜索，AI 是在消灭搜索。
必然结果
如果「意图直达」成为主流交互模式：① 服务商的获客逻辑将彻底改变——以前用户通过搜索/推荐找到你，以后是 AI 直接替你匹配用户，你只需要提供高质量的服务接口 ② SEO 将被 AEO（Answer Engine Optimization）取代——你的服务能不能被 AI 正确识别和推荐，比你的 App 在应用商店的排名重要一百倍 ③ 超级 App 的 AI 化将加速「围墙花园」——支付宝只推荐支付宝生态内的服务，微信只推荐微信生态内的服务。开放 vs 封闭，将是下一个十年的核心争论。

隐藏信号

Acti 键盘智能体来自一家新加坡初创——不是 Google、不是 Apple、不是微软。这意味着超级 App 的 AI 化和开放平台的 AI 化是两条不同的路。支付宝/微信走封闭生态，Acti 走设备层入口。谁能赢？我的判断是：最终胜出的是"那个你换不掉的东西"——对中文用户来说是微信/支付宝，对英文用户来说可能是手机操作系统。Acti 的键盘智能体很酷，但用户换掉它只需要 3 秒钟。

B1
一个人管 5 款产品，80% 时间不写代码——Every 的「复利工程」拆解 实战

来龙去脉

发生了什么：媒体软件公司 Every 公开其「复利工程」方法论——单人工程团队以 AI 辅助方式维护 5 款产品，80% 时间不写传统代码。核心是四步循环：Plan → Work → Review → Compound。Compound 步骤将每次解决问题的解法写入 CLAUDE.md 和 docs/solutions/，让 AI 不复用相同的错误，下次遇到同类问题直接调用已有方案。

为什么值得写：这是当前 AI 编程领域最被低估的能力——不是用 AI 写更多代码，而是用 AI 减少重复工作。mattpocockuk 的 writing-great-skills 指南也有类似洞察：AI Skill 的本质是「可复用的决策缓存」。

Every 复利工程 writing-great-skills

第一性原理拆解

底层结构
Every 的 Compound 步骤触及了一个被忽视的事实：AI 没有长期记忆。每次会话都是全新的开始——你昨天教会 AI 的东西，它今天全忘了。ClAUDE.md 的本质不是「文档」，而是 AI 的外部记忆。把解决问题的经验写进 AI 能在会话开头读取的文件中，等于在给 AI 装上一个"经历过"的假记忆。这是 AI 时代知识管理的终极形态——不是给人看的 wiki，而是给 AI 读的上下文。
物理/经济约束
为什么是现在出现了这种方法论？因为 模型的上文窗口在变长，但会话之间的隔离是结构性的。即使模型有 1M token 的上下文，每次新会话仍然从零开始。只要这个结构性隔离开在，Compound 式的「外部记忆」就是必需的基础设施。这不是某一款产品的功能缺失，是整个 LLM 架构的设计约束。

03Compound 实操：什么写进 CLAUDE.md，什么不该写

把 Claude Code 账单砍半：pxpipe 图像 token 压缩术 实战

来龙去脉

发生了什么：社区开发者发布了一个叫 pxpipe 的开源工具。它将系统提示、工具文档等密集文本渲染为 PNG 图像再发给模型——因为 API 对图像 token 的计费取决于像素尺寸，约 25k 文本 token 可压缩为约 2.7k 图像 token，端到端成本降低约 50%。

为什么值得写：花旗、Adobe 等企业已经开始限制员工用 AI 旗舰模型控制成本。AI 成本焦虑正在从个人蔓延到企业。省钱技巧在这个时间点有很强的传播力。

pxpipe 项目 企业限制 AI 用量

第一性原理拆解

底层结构
pxpipe 利用的是一个定价错位：API 对文本 token 按 token 数量计价（每 token 约 $X），对图像 token 按像素尺寸计价（每 1024² 像素约 $Y）。由于模型在视觉编码上的信息压缩比高于文本编码——一张 512×512 的 PNG 虽然包含大量像素，但模型看到的"信息量"远小于同样 token 数的文本——所以同一个 prompt 用图像传比用文本传便宜。这不是 bug，是 API 定价模型在跨模态场景下的不一致性。
必然结果
这种错位不可能长期存在——要么模型调整图像 token 定价，要么 pxpipe 被官方封杀。但 pxpipe 的出现本身揭示了一个更大的趋势：AI 成本套利将成为一门生意。就像金融市场的套利者会寻找定价不一致一样，AI 使用者会不断寻找 API 定价中的「漏洞」——缓存策略、批次优化、混合模型路由、token 压缩。这个市场最终会被定价优化中间层吃掉（类似 FinOps 之于云计算）。

03实操：从安装到看到省钱的完整流程

8 个必跑 Prompt Fable 5 做 SEO/GEO Fable 的判断力方法论

第一性原理拆解

底层结构
提示工程的核心不是"告诉 AI 做什么"，而是"定义 AI 的决策空间"。好的 Prompt 之所以有效，不是因为措辞巧妙，而是因为它缩小了模型的搜索空间。Fable 5 的 /goal 提示语本质上是在说：「你需要自主决定目标、设计实验、执行并汇报结果」——这不是"指令"，这是"角色+边界+输出格式"。模型越强，你需要给的具体指令越少，需要定义的边界和自由度越清晰。
必然结果
随着模型能力持续提升，提示工程将从「写指令」变成「设计系统」。未来的"优秀 prompt 工程师"实际上在做系统架构设计——定义决策边界、设计反馈回路、配置工具接口。这也是为什么 Fable 5 的 Prompt 技巧对 Sonnet 5 不完全适用——模型变了，决策空间的形状就变了。但方法论（定义边界、设定角色、明确输出）是可以迁移的。

设计工程师 Emil Kowalski 的「AI 审美」：让代码智能体学会 UI 品味 实战

Emil 的 UI Skill Claude Design 开源 writing-great-skills

第一性原理拆解

底层结构
Emil 做的事情本质上是在 把隐性知识转化为显性规则。设计审美是人类最难被 AI 复制的的能力之一——因为它通常靠"感觉"，而感觉无法被 hardcode。Emil 做的是把"感觉"分解为可验证的判断原则（动画有没有理由、响应速度够不够快、间距是否符合规范）。这不是教 AI「审美」，而是 给审美找到了可量化的代理指标。
必然结果

AI 能「看」视频了：browser-use 视频剪辑 Skill + claude-real-video 上手 实战

来龙去脉

发生了什么：browser-use 团队推出开源 Skill「video-use」，让 LLM 通过 ElevenLabs Scribe 将音频转写为文本，实现 AI 驱动的视频剪辑。另一个工具 claude-real-video 通过场景变化检测提取关键帧，让 LLM 基于视频画面（而非字幕）进行理解——支持 YouTube 和本地文件。此外，Wan Video 推出"音乐伴舞"功能（上传角色+选歌，AI 生成同步舞蹈），生数科技发布了 Vidu S1 实时交互视频模型。

video-use 剪辑 Skill claude-real-video Wan Video 伴舞

第一性原理拆解

底层结构
视频理解的瓶颈一直在多模态对齐——视觉信息（画面）和语言信息（字幕/音频）是两套不同语义空间的数据。以前的 AI"理解"视频本质上是只理解字幕文本——画面是盲区。claude-real-video 的关键帧提取+去重+转录方案，本质上是在建一座 廉价的多模态桥梁：不追求实时全量理解，而是通过关键帧采样把视觉信息降到文本模型能处理的信息密度。这不是最好的方案，但它是今天就能用的方案。
必然结果
多模态能力正在从"可以用"走向"足够好"的临界点。当 AI 真的能理解视频内容、生成视频内容、编辑视频内容，内容生产的边际成本会断崖式下降。不是从 100 降到 80，是从 100 降到 1。这跟 Midjourney 对平面设计的影响是一样的——原来需要一个团队才能做的事，一个人+AI 就足够了。对公众号而言，这意味着视频类内容的生产门槛大幅降低——纯文字号做视频内容的障碍正在消失。
